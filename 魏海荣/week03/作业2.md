- 正则表达式
    - 优点: 速度快、规则匹配精准
    - 缺点: 泛化能力差、需要手写正则匹配规则，语义相识的文本无法识别，同时维护成本高
- TF-IDF
    - 优点: 简单易实现，训练推理速度快。
    - 缺点: 只抓词频，依赖于词库，同样无法处理了语义相似、多义词。
- Bert
    - 优点: 适配场景广，可以很好的处理多义词/歧义句（有错别字的句子）/相似句，分类的准确率也高
    - 缺点: 需要占用计算资源，训练推理慢，并且需要数据资源（人工标注），对复杂句子和长句子的适配性差
- GPT-4
    - 优点: 适配场景比Bert更广，分类准确率高，可以处理一些训练少见的指令需求
    - 缺点: 依赖于网络速度，可以无法满足低延迟
